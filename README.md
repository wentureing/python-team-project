# Python Requests 小组项目

## 项目简介

这个项目的目的是通过分析 GitHub 上 Requests 项目的提交记录，来研究它的演化规律和开发模式。我们通过对提交数据进行统计和可视化，来了解项目的贡献者分布、提交类型以及随时间的变化趋势。

具体来说，我们分析了以下几个方面：
- **贡献者**：分析不同作者的提交情况，找到核心贡献者；
- **提交类型**：根据提交信息分类，了解项目中 bug 修复、新功能增加、依赖更新等类型的提交比例；
- **时间趋势**：分析项目每年的提交数量，观察项目的活跃度是否有变化。

通过这些分析，能帮助我们更好地理解开源项目的演化，以及它在不同时间段的开发模式。

---

## 数据说明

这个项目的数据来自 GitHub 上的 Requests 项目的 commit 记录，包含以下信息：
- **author**：提交者的名字
- **time**：提交的时间戳
- **message**：提交时的说明信息（简短描述）

数据以 CSV 格式存储，每一行代表一次提交。文件使用 UTF-8 编码，第一行是表头。

---

## 功能说明

### 1. 贡献者提交情况

- 统计并绘制提交次数最多的前 10 名贡献者，以此分析项目的贡献者结构。
- 我们提供了全周期的统计数据，同时也做了近五年和近两年的数据分析，帮助了解核心贡献者的变化。

### 2. 提交类型分析

- 根据提交信息的内容，将提交分为不同类型，包括：
  - **Merge PR**：合并请求
  - **Dependency**：依赖更新
  - **Release**：版本发布
  - **Bug Fix**：缺陷修复
  - **Feature**：新增功能
  - **Refactor**：代码重构
  - **Docs**：文档更新
  - **Test**：测试相关
  - **Maintenance**：维护性更新
  - **Other**：其他类型

### 3. 年度提交趋势

- 统计项目每年提交的数量，绘制年度活跃度趋势图。
- 你可以查看全项目周期的提交情况，也可以查看近五年和近两年的变化趋势。

### 4. 近五年与近两年的数据分析

- 我们对近五年和近两年的提交数据进行了独立分析，提供了更细化的贡献者和提交类型分析，帮助了解最近的项目发展情况。

## 使用说明

1. **准备数据文件**：
   - 将 GitHub 上 Requests 项目的 commit 数据导出为 CSV 文件，放入项目根目录。

2. **安装依赖**：
   项目需要matplotlib库来进行数据分析和可视化。可以使用以下命令安装：
   pip install matplotlib
