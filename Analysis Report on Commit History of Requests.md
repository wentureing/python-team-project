\# 三、核心分析结果



\## 3.1 贡献者生态：从创始人主导到社区协同与自动化维护

requests 贡献者生态呈现清晰的代际交接特征，逐步形成 “核心维护者 + 社区贡献 + 机器人自动化” 的协同模式。

全时段：项目创始人 Kenneth Reitz 以 3148 次提交居贡献榜首位，提交量超第二名 Cory Benfield（约 600 余次）的 5 倍。

近五年（2021-2025）：Kenneth Reitz 退出核心活跃行列，Nate Prewitt 成为核心维护者。

近两年（2024-2025）：贡献者结构趋于稳定，Nate Prewitt 持续领跑，dependabot \[bot] 稳居前三。



\## 3.2 提交活跃度：三阶段演进

爆发期（2011-2013 年）：年度提交量快速攀升。  

波动优化期（2014-2019 年）：提交量回落至 200-700 次 / 年。  

稳定维护期（2020 年至今）：提交量降至每年几十次，迭代节奏平缓。



\## 3.3 提交类型：开发导向到维护导向

全时段：Other 和 Merge PR 占比超过一半。  

近五年：Merge PR 成为主要提交类型。  

近两年：Merge PR 与 Dependency 为核心类型，Feature 提交量下降。
